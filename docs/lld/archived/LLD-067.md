# LLD-067: Clone-Last Architecture

**Issue:** #67
**Status:** Active

## 1. Problem

The pipeline requires local paths. URL targets (e.g., `https://github.com/org/repo`) silently produce zero results because `list_documentation_files()` returns an empty list for `target_type == "url"`. Users must manually clone repos before auditing them.

## 2. Solution: Clone-Last

Use GitHub Contents API for N0/N1 to discover and scan doc files remotely. Clone only in N5 when approved fixes exist.

```
N0 (Contents API: list files) -> N1 (Contents API: fetch + scan) -> N2 -> N3 -> N4 -> N5 (clone + fix)
```

## 3. Design

### 3.1 New Module: `github_api.py`

```python
# src/gh_link_auditor/github_api.py

class GitHubContentsClient:
    """Minimal GitHub Contents API client for file listing and fetching."""

    def __init__(self, token: str | None = None):
        # Uses GITHUB_TOKEN env var if token not provided
        # httpx client with auth header

    def list_doc_files(self, owner: str, repo: str) -> list[str]:
        """Recursively list doc files (.md, .rst, .txt, .adoc).

        Uses GET /repos/{owner}/{repo}/contents/{path} recursively.
        Returns sorted list of relative paths.
        """

    def fetch_file_content(self, owner: str, repo: str, path: str) -> str:
        """Fetch and decode file content.

        Uses GET /repos/{owner}/{repo}/contents/{path}.
        Decodes base64 content field.
        Returns file content as string.
        """
```

### 3.2 N0 Changes

`list_documentation_files()` currently returns `[]` for URL targets. Change to:

```python
if target_type == "url":
    owner, repo = repo_name.split("/")
    client = GitHubContentsClient()
    return client.list_doc_files(owner, repo)
```

Add `repo_owner` and `repo_name_short` to state (extracted from URL in N0).

### 3.3 N1 Changes

`_extract_urls_from_file()` currently reads via `Path(filepath).read_text()`. Add a content-fetching abstraction:

```python
def _read_file_content(
    filepath: str,
    target_type: str,
    repo_owner: str = "",
    repo_name_short: str = "",
) -> str:
    if target_type == "local":
        return Path(filepath).read_text(encoding="utf-8", errors="replace")
    client = GitHubContentsClient()
    return client.fetch_file_content(repo_owner, repo_name_short, filepath)
```

Pass `target_type`, `repo_owner`, `repo_name_short` from state through `run_link_scan()` and into `_extract_urls_from_file()`.

### 3.4 N5 Changes

```python
    # Remap source_file paths: relative -> absolute in clone dir

```

Reuse the clone pattern from `docfix_bot/git_workflow.py`.

### 3.5 State Changes

Add to `PipelineState`:
```python
repo_owner: str      # "org" extracted from URL
repo_name_short: str # "repo" extracted from URL
clone_dir: str       # temp dir path if cloned (N5 only)
```

## 4. API Rate Budget

- N0: ~5-10 calls (recursive directory listing)
- N1: ~30 calls (one per doc file)
- Total: ~40 calls/repo, well within 5,000/hr authenticated limit
