# LLD-036: Stargazer-Targeted Discovery Mode

**Issue:** #36
**Status:** Approved
**Date:** 2026-03-07

## Overview

Add stargazer harvesting to repo_scout: given seed repos, fetch their stargazers, discover repos those users own, and feed them into the existing pipeline.

## Changes

### 1. models.py

Add `STARGAZER_TARGET = "stargazer_target"` to `DiscoverySource` enum.

### 2. github_client.py -- Two new methods

**`get_stargazers(owner, repo, max_count=100) -> list[str]`**
- `GET /repos/{owner}/{repo}/stargazers?per_page=100&page=N`
- Returns list of usernames (login field)
- Stops at `max_count` or end of pages

**`get_user_repos(username) -> list[RepositoryRecord]`**
- `GET /users/{username}/repos?type=owner&sort=updated&per_page=100`
- Filters: `fork=True`, `archived=True`
- Source: `STARGAZER_TARGET`
- Metadata: `pushed_at`, `has_issues`, `language`

### 3. stargazer_harvester.py -- New module

```python
def harvest_from_stargazers(
    seed_repos: list[str],
    github_client: GitHubClient,
    max_stargazers: int = 100,
    max_repo_age_months: int = 6,
    progress_callback: Callable | None = None,
) -> list[RepositoryRecord]:
```

- Parse owner/repo from seed strings
- Fetch stargazers per seed, track visited usernames
- Fetch owned repos per stargazer, filter by recency
- Helper: `_is_recently_active(pushed_at, max_age_months) -> bool`

### 4. cli.py -- Three new args

- `--seed-repos`: nargs="*", default=[]
- `--max-stargazers`: int, default=100
- `--max-repo-age-months`: int, default=6

Wire as Source 4 after LLM suggestions.
