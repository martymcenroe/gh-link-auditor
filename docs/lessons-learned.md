# Lessons Learned — gh-link-auditor

| Date | Lesson | Rule/Action |
|:-----|:-------|:------------|
| 2026-05-12 | **`gh issue list --search "..."` ignores `--repo` and returns global results.** Tried `gh issue list --state all --search "..." --repo owner/repo` for a scoped search and got back 30 issues from completely unrelated repos. | **Don't combine `--search` with `--repo` expecting scoping. Use `gh issue list --state all --limit N --repo OWNER/REPO --json ...` and post-filter the titles client-side, OR use the search API with explicit `repo:OWNER/REPO` in the query string.** |
| 2026-05-12 | **Background pytest piped to a file looks hung — Python's stdio uses full buffering when stdout isn't a tty.** Spent ~15 min wondering if `pytest -q ... > log &` had stalled at 93%; it was running fine, just buffering all dots until summary. | **For background pytest runs that need progress visibility: set `PYTHONUNBUFFERED=1`, use `-u`, or use `-v` (line-buffered formatter). Foreground pytest in the harness streams correctly; the harness's auto-backgrounding handles the output file already.** |
| 2026-05-12 | **MEMORY.md test-count milestones can be unverifiable.** Memory had "1,705 tests after #148-#150" but `poetry run python -m pytest --version` failed with `No module named pytest` — tests had been running via base Python's globally-installed pytest the whole time, not the project venv. | **Before trusting a memory's test-count milestone, verify `poetry run python -m pytest --co` actually collects in the current venv. If `pytest` isn't in pyproject deps, the milestone describes a state that no longer reproduces.** |
| 2026-05-12 | **Time-boundary tests with hardcoded ISO dates rot.** `test_boundary_exact_age` hardcoded `2025-10-01` as "within the 6-month window" at write-time; today (224 days later) it's outside the 180-day window so the function correctly returns False and the test fails. | **In time-boundary tests, compute timestamps relative to `datetime.now`: `(datetime.now(timezone.utc) - timedelta(days=N)).strftime(...)`. Hardcoded dates ARE time bombs. Centralize via a module-level constant if reused.** |
| 2026-05-12 | **"Tests exist" ≠ "tests run at merge gate."** Repo had 1,749 tests passing locally but `.github/workflows/lint.yml` was the only gating workflow and ran zero tests. The repo's own "≥95% test coverage is mandatory" rule was unenforced. | **When auditing a repo, always check `.github/workflows/*` for what's actually gated. Treat the gate as the rule, not the README, the CLAUDE.md, or the dev's intent. Add a `Test` workflow whenever you find tests-without-gate.** |
| 2026-05-12 | **`.claude/worktrees/agent-*` are submodule gitlinks (mode 160000), not real directories.** Tried `rmdir` on an empty one and got "Device or resource busy"; only `git merge --ff-only` revealed they were `create mode 160000` entries. | **Before trying to filesystem-delete anything under `.claude/worktrees/` (or any agent-managed worktree path), run `git ls-tree HEAD .claude/worktrees/` first. If it shows commit gitlinks, the dirs are submodule references — clean via `git rm --cached` + commit, not `rmdir`.** |
| 2026-05-13 | **Sites can return 404 to HEAD but 200 to GET.** Microsoft Marketplace does this as anti-crawler defense. | **For any 4xx commonly used for bot-blocking (403, 404, 405), default to "try GET once" before declaring dead. Only 410 is safely terminal on HEAD.** |
| 2026-05-13 | **A successful redirect chain from a dead URL means the URL works.** SO short URLs `/a/N` redirect to long form — both work; "fix" is cosmetic. | **In any chain-following investigation: if the original URL responds (even 30x), it isn't broken — don't propose a candidate.** |
| 2026-05-13 | **Cloudflare-fronted hosts can return None status (TCP reset) not 403.** Pythonjobshq.com bypasses 403-only stealth fallback. | **Bot-blocking has multiple layers (HTTP 4xx, TCP reset, TLS challenge). Stealth fallback should check all, gated by a CDN-IP heuristic.** |
| 2026-05-13 | **GitHub's auto-close from PR body only fires for the FIRST `#N` after `Closes`.** "Closes #194 #195 #196" closed #194 only. | **Write one `Closes #N` per line in PR bodies. The keyword needs to repeat. Same for `Fixes`, `Resolves`.** |
| 2026-05-13 | **Polite-but-firm maintainer rejection misses `hostile_classifier`.** Pallets/flask davidism's "Happy to update this, but please do not use genAI…" matched no rude phrase. | **Different rejection classes need different classifiers. Hostile (#178) catches rude; anti-AI (#200) catches polite-firm. Use distinct `source=` values.** |
| 2026-05-13 | **JS-challenge pages return 403 to bots but load fine in real Chrome with stealth.** Career project's TypeScript Playwright + stealth pattern ported directly to Python with `playwright-stealth.Stealth().apply_stealth_sync(page)`. | **Use `playwright` with `channel="chrome"` — NOT `playwright install chromium`. No 150MB download. The bot-detection delta between bundled-Chromium and real-Chrome is significant.** |
| 2026-05-13 | **Operator-discovered false positives during HITL are the highest-signal labeled data.** Each one is a confirmed pipeline-disagrees-with-reality case the operator just verified. | **Build an `[l]ive` flag into any HITL prompt that asks a human to validate machine output. Print at end-of-session; persist to DB if possible. Mining false-positive patterns is how the pipeline improves.** |
